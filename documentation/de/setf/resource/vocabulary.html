This file defines a representation for RDF vocabularies.
The vocabulary operators map type/class models between CLOS and RDF schemas. The CLOS representation includes class instances and property accessors. The RDF representation is retrieved from a triple store. The vocabulary also includes an optional identifier map to specify the relations between Lisp symbols and RDF uri. Notes on the 'canonical' vocabulary namespace names: The XML Schema namespaces are 'http://www.w3.org/2001/XMLSchema' and 'http://www.w3.org/2001/XMLSchema-datatypes'[1], depending on whether the circumstances involve schema aspects inaddition to the datatype terms. Note that, even though these lack a fragment separator character, the standard prefix binding is 'xsd=http://www.w3.org/2001/XMLSchema#'[2]. The original OWL recommendation, on the other hand, avails itself of both approaches. On one hand, the general discussion of namespaces demonstrates the presence of fragment separator characters in the vocabulary namespace[3], but on the other the discussion of extensions[4], suggests that fragements are integral entites which _include_ the separator character. While this may be just the result of editorial oversight, it demonstrates that the operators can perform neither a strict url merge, not a simple string catenation if they are to produce the expected results from the widely varies input arguments. --- [1]: http://www.w3.org/TR/2001/REC-xmlschema-2-20010502/#namespaces [2]: http://www.w3.org/TR/2004/REC-rdf-mt-20040210/ [3]: http://www.w3.org/TR/2004/REC-owl-guide-20040210/#Namespaces [4]: http://www.w3.org/TR/2004/REC-owl-guide-20040210/#DefiningSimpleClasses

Class
vocabulary
( standard-object )
A vocabulary represents an RDF schema in Lisp terms and CLOS class definitions. It comprises a set of class definitions, which implicitly define the schema's predicates, and an optional map between Lisp names and the schema's URI. A vocabulary can be constructed based on an RDF representation as mediated by a triple store, or it can be cached as an explicit definition in the file system.
name
[t]
uri
[t]
The vocabulary base uri identifies the 'namespace' which comprises its terms and, by default, locates the schema declarations.
identifier-map
[t]
An a-list of maps between model symbols and resource URI namestrings for the external schema definition. The default value is derived from a combination of the vocabulary uri, the values of the :identifiers and :identifier-map initialization arguments. The former is a list of symbols or strings, of which the latter are interned in the respective package.
