de.setf.resource
`de.setf.resource` implements a transparent projection for RDF repositories as CLOS models. The implementation relies on the CLOS-MOP support for augmented slot access mechanisms to equate class/slote structure with a resource-node[0] oriented view of RDF graphs. It includes interfaces to wilbur[1,2] and allegro-graph[3] repositories and permits both resident and remote repositories. The core implementation involves three classes - the resource-metaclass and resource-class metaclasses, and the resource-object class. These extend the standard-class and standard-object implementations to identify concrete classes with repositories, classes with URI realms, instances with individual URI, and slots with property and computed predicates. Extensions to the instantiation and slot access protocols combine with transaction support to implement instance life-cycles analogous to the JDO persistence life-cycle[4].
